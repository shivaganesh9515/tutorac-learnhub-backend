Authentication & Roles

JWT-based Authentication

  1. Client sends email + password.
  2. Password is hashed using bcrypt.
  3. On success, a JWT is signed with userId and role.
  4. Client stores token (e.g., in memory / secure storage).
  5. All protected routes require Authorization: Bearer <token>.

Role-based Access Control (RBAC)

  • student â€“ basic access, consumes content.
  • instructor â€“ can create/update courses.
  • admin â€“ manage users & roles.

Middleware used:

  • auth â€“ verifies JWT & attaches req.user
  • requireRole("admin") â€“ ensures logged-in user has the required role
Example protected route 
router.post(
  "/course/create",
  auth,
  requireRole("instructor"),
  (req, res) => {
    res.json({ message: "Course created successfully" });
  }
);
